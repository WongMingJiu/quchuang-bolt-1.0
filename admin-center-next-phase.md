# 管理中心下一阶段开发清单

## 一、目标定义

当前管理中心已经具备：

- 菜单入口
- 管理中心页面
- 团队 / 个人 / 权限额度 三个 tab
- 基础抽屉交互
- 后端接口骨架
- 数据库 migration 基础

下一阶段的目标是：

> **从 mock 骨架版升级成真实数据驱动的管理后台可用版**

## 二、阶段划分

建议下一阶段按 4 个阶段推进：

1. **真实数据接入**
2. **详情与编辑联调**
3. **数据口径与权限完善**
4. **交互与体验收尾**

## 阶段 3：数据口径与权限完善

### Task 3.1 明确“次数”和“token”双额度口径
你已经确认：
- 两个都要

所以后续要统一所有页面字段表达：

#### 团队页
- 月剩余次数
- 月剩余 token

#### 个人页
- 月剩余次数
- 月剩余 token

#### 权限页
- 月总次数 / 已用次数 / 剩余次数
- 月总 token / 已用 token / 剩余 token

### Task 3.2 扩展 `model_quota_assignments`
当前表还偏简。

#### 建议新增字段
- `monthly_call_quota`
- `used_call_quota`
- `remaining_call_quota`
- `monthly_token_quota`
- `used_token_quota`
- `remaining_token_quota`

#### 目标
让“次数”和“token”两套口径都能被后台真正管理。

### Task 3.3 团队成员统计接口补真实成员维度
当前团队成员统计接口还是“当前用户占位”。

#### 下一步要做
把它接到你现有的用户 / 角色体系里，真正返回：
- 成员名称
- 角色
- 成员维度统计

### Task 3.4 预估费用口径确认

- 是否按模型定价
