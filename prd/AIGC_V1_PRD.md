## 2. 一句话概述

> 为了帮助信息流广告素材团队和内容运营团队提升视频生产效率、沉淀素材资产，本需求将提供基于 Seedance 的多模式视频生成、素材上传引用、资产管理与管理中心能力，从而形成可复用、可追踪、可筛选的 AIGC 视频生产闭环。

## 3. 背景与问题

### 3.1 业务背景

当前仓库已经实现了一个较完整的 AIGC 视频创作产品雏形，覆盖创作工作台、历史记录、资产中心、管理中心、Seedance 生成接口、Supabase 数据存储与素材管理能力。
从功能设计可以看出，这个产品并非面向单次体验式生成，而是服务于内容团队的持续生产场景。

对于信息流广告素材团队和内容运营团队而言，视频创作通常具备以下特点：

- 同一业务主题需要持续产出多条素材
- 同一条素材需要频繁迭代提示词、比例、时长、音频、水印等参数
- 团队需要保留历史生成结果，快速复用有效素材
- 素材不仅要“能生成”，还要能被筛选、收藏、标注和沉淀

当前产品正是在这样的背景下，试图把“提示词输入 → 参考素材上传 → 视频生成 → 结果回看 → 资产沉淀”串成统一闭环。

**依据：** `README.md`、`src/App.tsx`、`src/pages/CreationPage.tsx`、`src/pages/AssetsPage.tsx`、`src/pages/AdminCenterPage.tsx`

### 3.2 用户问题

当前信息流广告素材团队和内容运营团队在使用 AIGC 进行视频生产时，通常会遇到以下问题：

1. **创作链路割裂**
   - 提示词、参考素材、生成参数和结果分散在不同工具或上下文中
   - 同一条视频的生成过程难以回溯和复用

2. **素材难沉淀**
   - 生成结果数量多，但缺少统一的筛选、收藏、标注和资产分类机制
   - 团队难以快速识别哪些素材可继续复用

3. **生产效率不稳定**
   - 每次生成都需要重复填写参数和重新组织参考素材
   - 缺少基于历史任务快速回填和再生成的能力

4. **团队管理能力不足**
   - 随着使用规模扩大，需要对模型权限、额度、任务消耗和用户使用情况进行管理
   - 若缺少管理中心，产品难以支撑团队化使用

## 7. 业务流程

### 7.1 当前流程

1. 用户进入创作工作台。
2. 填写提示词，选择生成模式与相关参数。
3. 根据模式上传参考素材或直接发起文生视频任务。
4. 前端调用 `POST /api/generate` 创建任务。
5. 任务进入数据库并开始轮询状态。
6. 前端调用 `GET /api/generate/:id` 查询任务进度。
7. 任务完成后，结果写入数据库并展示在历史记录和资产中心。
8. 用户可对历史记录进行回填、再生成、删除、收藏。
9. 用户可在资产中心查看、下载、标注结果资产。
10. 管理员可在管理中心查看团队/个人使用情况并管理模型额度。

### 7.2 目标流程

1. 用户围绕具体业务场景快速组织提示词和参考素材。
2. 系统帮助用户标准化生成输入，缩短提交成本。
3. 用户在多轮生成中不断迭代更优结果。
4. 有价值的结果自动沉淀为资产，并被分类、收藏、标注。
5. 管理者通过管理中心控制资源使用与团队效率。

### 7.3 流程图

```text
用户进入创作工作台
    ↓
填写提示词并选择模式/参数
    ↓
上传或引用参考素材
    ↓
提交生成任务
    ↓
系统轮询任务状态
    ↓
生成完成 / 失败反馈
    ↓
结果进入历史记录与资产中心
    ↓
用户收藏 / 标注 / 下载 / 再生成
    ↓
管理员在管理中心查看任务与额度情况
```

## 8. 数据结构

### 8.1 核心实体总览

当前仓库中，围绕本次需求最核心的实体是 **生成任务（Generation）**。从前端类型、数据库结构、接口请求体和资产中心展示逻辑来看，产品的主要业务对象都围绕该实体展开，并通过字段区分其在创作、结果资产、参考素材、IP老师资产等不同场景下的角色。

除 `Generation` 外，产品中还存在一些明确但相对次级的结构化对象，例如：

- `media_uploads`：生成任务关联的参考素材数组
- Prompt Expansion Item：提示词扩写候选项
- Provider Task：外部 Seedance/Ark 任务返回体
- Quota Assignment：模型权限与额度配置项（当前前端管理页已存在结构和骨架）

### 8.2 实体一：Generation（生成任务 / 资产主实体）

**实体说明**  
`Generation` 是本产品最核心的业务实体。它既承载一次视频生成任务的输入参数，也承载任务执行状态、结果文件地址、素材引用、收藏状态和资产分类信息。在产品上，它同时服务于：

- 创作工作台的任务创建
- 历史记录的回看与再生成
- 资产中心的展示、筛选、预览、标注与下载
- 管理中心的统计基础数据来源

**主要字段**

|字段名|类型|说明|
|---|---|---|
|id|uuid|主键，任务 ID|
|prompt|text|用户输入提示词|
|mode|text|生成模式|
|model|text|当前使用模型，前端默认 `seedance2.0`|
|category|text|所属品类|
|storyboard_type|text|分镜类型|
|aspect_ratio|text|视频比例|
|duration|integer|视频时长（秒）|
|generate_audio|boolean|是否生成音频|
|watermark|boolean|是否带水印|
|status|text|任务状态|
|video_url|text|视频结果地址|
|thumbnail_url|text|缩略图地址|
|last_frame_url|text|末帧图地址|
|is_favorited|boolean|是否收藏|
|media_uploads|jsonb|参考素材数组|
|provider|text|生成供应商，如 `seedance`|
|provider_task_id|text|外部任务 ID|
|error_message|text|失败错误信息|
|completed_at|timestamptz|任务完成时间|
|created_at|timestamptz|任务创建时间|
|asset_category|text|资产分类，前端已使用 `creative / reference / ip_teacher`|
|asset_media_type|text|资产媒介类型，前端已使用 `image / video / audio`|
|ip_asset_type|text|IP老师子类，前端已使用 `persona / scene`|
|usability_status|text|可用性标注状态|
|usability_reason_tags|text[] / json 数组|可用性标注原因标签|
|usability_note|text|可用性标注备注|

**依据：** `database-structure.md`、`api-reference.md`、`src/pages/CreationPage.tsx`、`src/pages/AssetsPage.tsx`、`src/components/annotation/UsabilityAnnotationModal.tsx`

## 9. 产品结构

### 9.1 产品结构概览

本产品当前可以按四层结构理解：

1. **表现层**：用户直接操作和看到的前端页面与交互组件
2. **应用层**：围绕创作、资产、管理等业务流程的页面逻辑、状态管理和调用编排
3. **数据层**：Supabase 数据表、存储桶、接口返回结构以及前端数据模型
4. **底层服务层**：Seedance / 火山方舟、Vercel Functions、本地 Prompt Expansion API、Supabase 服务能力

该结构说明产品不是单页生成工具，而是围绕视频创作、资产沉淀和运营治理搭建的一体化工作平台。

### 9.2 表现层

表现层主要承载用户界面、交互反馈和页面切换，当前可确认包括：

|创作工作台|`src/pages/CreationPage.tsx`、`src/components/creation/*`|承载提示词输入、素材上传、参数配置、生成提交|
|历史记录区|`src/components/history/*`|承载历史任务展示、回填、再生成、收藏、删除|

### 9.5 底层服务层

底层服务层负责真正执行生成、扩写、存储、查询与部署相关能力，是整个产品运行的基础设施支撑。

|服务|职责|说明|
|---|---|---|
|Seedance / 火山方舟|执行视频生成任务|生成能力核心外部服务|
|Vercel Functions|承载 `/api/generate`、`/api/generate/:id`、`/api/prompt-expand` 等服务端接口|前端与外部能力之间的后端桥梁|
|Supabase Database|存储生成任务与相关业务数据|核心结构化数据存储层|
|Supabase Storage|存储上传素材并提供公开 URL|支撑参考素材注入生成链路|
|Prompt Expansion 服务|扩写提示词草稿为候选文案|支持创作输入增强|
|本地 Prompt Expansion API|开发模式下替代线上扩写服务|降低本地联调成本|
|Vite 前端运行时|承载页面渲染与前端交互逻辑|产品主要前端容器|

### 9.6 分层关系说明

```text
表现层
  - 创作页 / 资产页 / 管理中心 / 历史记录 / 标注弹层
      ↓
应用层
  - 生成任务编排 / 提示词扩写 / 资产筛选 / 标注保存 / 配额编辑
      ↓
数据层
  - generations 表 / media_uploads / 可用性标注字段 / generation-assets
      ↓
底层服务层
  - Seedance / Ark / Vercel Functions / Supabase / Prompt Expansion 服务
```

### 9.7 当前产品结构的特点

- **以生成任务为中心**：创作、历史、资产都围绕 `Generation` 实体展开
- **前后端边界清晰**：前端负责交互与状态编排，后端 API 负责生成与轮询
- **数据与资产结合紧密**：数据库记录任务状态，存储桶承载素材文件
- **具备运营治理雏形**：除创作链路外，已经延伸出资产标注和管理中心能力

## 10. 功能需求

## 10.1 创作工作台

### 10.1.1 功能说明

该功能用于承载视频生成的主输入流程，用户可以通过填写提示词、选择生成模式和配置参数完成 Seedance 任务提交。

### 10.1.2 使用角色

|角色|是否可用|权限说明|
|---|---|---|
|广告素材团队成员|是|可输入提示词并发起生成|
|内容运营团队成员|是|可上传素材并生成内容|
|管理员|是|具备使用权限|
|普通访客|待确认|当前仓库未明确访客逻辑|

### 10.1.3 前置条件

|条件|说明|
|---|---|
|前端可访问生成接口|否则无法创建任务|
|环境变量已配置|正式模式需配置 Supabase 和 Ark/Seedance 环境|
|符合模式要求的素材已准备好|特定模式下素材数量/类型有限制|

### 10.1.4 主流程

1. 用户输入提示词。
2. 用户选择生成模式。
3. 用户配置参数（品类、分镜、比例、时长、音频、水印等）。
4. 用户按需上传参考素材。
5. 系统校验输入是否合法。
6. 校验通过后提交生成任务。

### 10.1.5 规则说明

|规则项|说明|
|---|---|
|模式校验|不同模式要求不同的素材数量与类型|
|素材数量上限|最多支持 9 个文件|
|时长范围|4~15 秒|
|文生视频|无需上传素材|
|图生视频-首尾帧|必须上传 2 张图片|
|图生视频|仅支持图片|
|全能参考|支持图片、视频、音频混合参考|

### 10.1.6 页面字段

|字段名称|字段类型|是否必填|默认值|校验规则|说明|
|---|---|---|---|---|---|
|提示词|文本|是|空|不能为空|生成核心输入|
|模式|单选|是|text-to-video|枚举值限制|决定素材和请求结构|
|模型|文本/选择|是|seedance2.0|待确认|当前前端默认使用 seedance2.0 |
|所属品类|选择|是|太极|枚举值限制|用于业务场景分类|
|分镜类型|选择|是|口播类|枚举值限制|用于内容结构分类|
|画面比例|选择|是|16:9|枚举值限制|控制输出比例|
|视频时长|数字|是|10|4~15 秒|控制结果时长|
|是否有声音|布尔|是|true|无|控制生成音频|
|是否带水印|布尔|是|false|无|控制输出带水印与否|

### 10.1.7 状态说明

|状态|触发条件|用户可见表现|后续动作|
|---|---|---|---|
|编辑中|用户正在填写表单|可编辑各项输入|可提交|
|校验失败|输入不符合规则|展示错误提示|需修正后再提交|
|提交中|已发起生成请求|按钮进入 generating 状态|等待接口返回|
|生成中|任务创建成功|显示任务中状态|轮询任务结果|
|完成|任务成功|结果进入历史与资产中心|可预览/下载/标注|
|失败|任务失败|显示错误消息|可修改后重试|

### 10.1.8 异常场景

|异常场景|系统处理|用户提示|
|---|---|---|
|提示词为空|阻止提交|请输入提示词|
|素材超出上限|阻止提交|上传素材不能超过 9 个|
|模式与素材类型不匹配|阻止提交|显示对应模式校验文案|
|接口创建失败|展示错误|创建任务失败，请稍后重试|
|轮询失败|停止轮询|轮询生成状态失败|

### 10.1.9 验收标准

|编号|验收标准|
|---|---|
|AC1|Given 用户输入完整提示词和合法参数，When 点击生成，Then 系统应创建视频生成任务|
|AC2|Given 用户选择图生视频-首尾帧模式，When 上传素材数量不是 2 张图片，Then 系统应阻止提交并提示错误|
|AC3|Given 任务创建成功，When 前端轮询到完成状态，Then 历史记录中应出现生成结果|

## 10.2 提示词扩写

### 10.2.1 功能说明

该功能用于帮助用户把简短的提示词草稿扩展为更完整的 Seedance 候选提示词，从而降低创作输入门槛，提升生成文案的完整度和可用性。

### 10.2.2 使用角色

|角色|是否可用|权限说明|
|---|---|---|
|广告素材团队成员|是|可输入草稿并生成候选提示词|
|内容运营团队成员|是|可快速补全创意文案|
|管理员|是|具备使用权限|

### 10.2.3 前置条件

|条件|说明|
|---|---|
|用户输入提示词草稿|空输入时不允许扩写|
|提示词扩写接口可访问|依赖 `/api/prompt-expand` 或本地扩写 API|
|扩写服务环境变量已配置|正式环境需配置扩写服务参数|

### 10.2.4 主流程

1. 用户输入一个简短提示词草稿。
2. 用户点击“扩写提示词”。
3. 前端调用 `/api/prompt-expand` 请求候选结果。
4. 系统返回 1~3 条扩写候选。
5. 用户点击“使用”，将候选提示词覆盖正式提示词输入框。

### 10.2.5 规则说明

|规则项|说明|
|---|---|
|空输入限制|未输入草稿时不给发起扩写|
|返回结果数量|目标为 1~3 条候选结果|
|使用方式|点击“使用”后覆盖正式提示词输入框|
|本地开发模式|支持通过轻量本地 API 代理扩写接口|
|错误兜底|扩写失败时返回明确报错文案|

### 10.2.6 状态说明

|状态|触发条件|用户可见表现|后续动作|
|---|---|---|---|
|未开始|尚未输入草稿或未点击扩写|显示默认说明|可继续输入|
|扩写中|已发起扩写请求|按钮显示“扩写中...”|等待接口返回|
|扩写成功|返回候选结果|展示候选卡片|可选择“使用”|
|扩写失败|接口报错或无结果|显示错误提示|可重新尝试|

### 10.2.7 异常场景

|异常场景|系统处理|用户提示|
|---|---|---|
|未输入草稿|阻止请求|请先输入待扩写的提示词草稿|
|接口请求失败|展示错误|提示词扩写失败，请稍后重试|
|服务无候选结果|视为失败|扩写服务未返回可用候选结果，请稍后重试|

### 10.2.8 验收标准

|编号|验收标准|
|---|---|
|AC1|Given 用户输入提示词草稿，When 点击“扩写提示词”，Then 系统应返回 1~3 条候选结果|
|AC2|Given 候选结果已返回，When 用户点击“使用”，Then 正式提示词输入框应被对应候选内容覆盖|
|AC3|Given 扩写接口异常，When 请求失败，Then 系统应展示可理解的错误提示|

## 10.5 可用性标注

### 10.5.1 功能说明

该功能用于对创作结果进行结构化质量判断，记录结果是否可直接使用、是否需要优化、或是否不可用，并支持补充原因标签和备注说明，从而支撑素材筛选、沉淀与后续复用。

### 10.5.2 使用角色

|角色|是否可用|权限说明|
|---|---|---|
|内容运营团队成员|是|可对创作资产进行标注|
|资产管理角色|是|可作为素材沉淀与筛选依据|
|广告素材团队成员|是|可在下载或复盘前完成标注|

### 10.5.3 前置条件

|条件|说明|
|---|---|
|资产为创作资产|当前主要作用于创作资产，不适用于其他资产分类|
|生成结果已存在|需存在可被评价的创作结果|
|标注弹层可访问|依赖前端可用性标注交互组件|

### 10.5.4 主流程

1. 用户在资产中心查看创作资产。
2. 用户点击“去标注”或“修改”进入可用性标注弹层。
3. 用户选择主状态：可用 / 可优化 / 不可用。
4. 若选择“可优化”或“不可用”，需至少选择一个原因标签。
5. 用户可补充备注说明。
6. 系统保存标注结果，并在资产卡片和筛选条件中回显。
7. 若用户从下载动作触发标注，保存后可继续下载。

### 10.5.5 规则说明

|规则项|说明|
|---|---|
|主状态|支持可用、可优化、不可用三类主状态；未标注时视为待标注|
|原因标签|“可优化”与“不可用”状态下需至少选择一个原因标签|
|备注长度|备注最多 200 字|
|下载前标注|待标注创作资产在下载前可先进入标注流程|
|回显逻辑|标注结果应回显在资产卡片与筛选条件中|

### 10.5.6 状态说明

|状态|触发条件|用户可见表现|后续动作|
|---|---|---|---|
|待标注|未完成标注|资产卡片显示“待标注”|可进入标注|
|可用|结果质量达标|显示“可用”标签|可直接沉淀与复用|
|可优化|结果基本可用但需调整|显示“可优化”标签|可作为二次迭代素材|
|不可用|结果不建议继续使用|显示“不可用”标签|用于筛除低质量结果|

### 10.5.7 异常场景

|异常场景|系统处理|用户提示|
|---|---|---|
|未选择主状态|阻止保存|需完成标注后才能保存|
|可优化/不可用未选原因标签|阻止保存|请选择至少一个原因标签|
|保存失败|保留当前输入|提示保存失败并允许重试|

### 10.5.8 验收标准

|编号|验收标准|
|---|---|
|AC1|Given 用户打开标注弹层，When 选择“可优化”或“不可用”，Then 系统应要求至少选择一个原因标签|
|AC2|Given 用户保存标注结果，When 返回资产列表，Then 卡片上应显示对应标注状态|
|AC3|Given 创作资产处于待标注状态，When 用户从下载入口触发标注并保存，Then 系统应允许继续下载|

## 10.6 管理中心

### 10.6.1 功能说明

### 10.6.4 主流程

1. 用户进入管理中心。
2. 切换团队仪表盘 / 个人仪表盘 / 模型权限与额度。
3. 管理员查看使用情况并配置用户/角色级额度。

### 10.6.5 规则说明

|规则项|说明|
|---|---|
|时间范围切换|支持当天、24小时、7天、30天、60天|
|管理维度|团队、个人、额度管理|
|额度对象|支持按用户或角色配置|
|可用模型|支持配置特定模型可用范围|

### 10.6.6 验收标准

|编号|验收标准|
|---|---|
|AC1|Given 管理员进入管理中心，When 切换标签页，Then 应能查看团队仪表盘、个人仪表盘和额度管理内容|
|AC2|Given 管理员编辑某个用户/角色额度，When 保存成功，Then 新额度应生效并可回看|
|AC3|Given 管理员选择时间范围，When 切换时间范围，Then 仪表盘数据应按范围刷新|
