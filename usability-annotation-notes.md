# 可用性标注功能说明文档

## 一、功能目标

“可用性标注”能力用于帮助用户在浏览生成结果后，对结果进行主观质量判断，并将该判断与结果卡片绑定。

该能力的核心目标是：

1. 让用户可以快速标注某条生成结果是否可用
2. 在结果列表中持续回显标注状态
3. 强制要求用户在下载前完成标注
4. 保持交互轻量，不破坏当前卡片和播放器结构

## 二、适用范围

当前支持以下两个区域：

- 历史记录区
- 资产页

对应文件：
- `src/components/history/HistoryCard.tsx`
- `src/pages/AssetsPage.tsx`

## 三、功能结构

该能力由三部分组成：

### 1. 卡片内轻量状态展示
每张结果卡片上会显示：

- 右上角状态 badge
- 一行“可用性标注”状态行
- 一个操作入口：
  - `去标注`
  - 或 `修改`

### 2. 下载前强拦截
如果当前结果还没有标注：

- 用户点击下载按钮
- 不会直接下载
- 而是弹出“可用性标注”弹层
- 标注完成后，才继续下载

### 3. 标注弹层
统一组件：
- `src/components/annotation/UsabilityAnnotationModal.tsx`

功能：
- 展示当前结果基本信息
- 选择可用性状态
- 选择原因标签
- 填写备注
- 保存标注
- 或在下载拦截场景中“保存并下载”

## 四、标注状态定义

当前支持 4 种状态：

### 1. `pending`
中文展示：
- `待标注`

说明：
- 默认状态
- 新生成结果默认都是待标注

### 2. `usable`
中文展示：
- `可用`

说明：
- 结果达标，可直接使用

### 3. `optimizable`
中文展示：
- `可优化`

说明：
- 结果基本可用，但还需要进一步调整

### 4. `unusable`
中文展示：
- `不可用`

说明：
- 当前结果不建议继续使用

## 五、卡片上的交互表现

### 未标注时
卡片会显示：

- badge：`待标注`
- 状态行：
  - `可用性标注：待标注`
- 右侧按钮：
  - `去标注`

### 已标注时
卡片会显示：

- badge：`可用 / 可优化 / 不可用`
- 状态行：
  - `可用性标注：可用 / 可优化 / 不可用`
- 右侧按钮：
  - `修改`

## 六、下载按钮逻辑

### 1. 未标注
- 点击下载
- 不直接下载
- 弹出标注弹层
- 标注完成后自动继续下载

### 2. 已标注
- 点击下载
- 正常下载，不再拦截

## 七、标注弹层结构

弹层组件：
- `UsabilityAnnotationModal`

### 顶部区域
显示：
- 当前结果预览图
- prompt 摘要
- 任务基本信息标签：
  - 比例
  - 时长
  - 品类
  - 分镜类型

### 中部表单区域
#### 1. 主状态（必选）
- 可用
- 可优化
- 不可用

#### 2. 原因标签
当前会根据主状态显示标签：

**可优化**
- 动作不自然
- 节奏不稳
- 画面细节欠佳
- 构图可优化
- 人物表现可优化

**不可用**
- 画面崩坏
- 人物异常
- 风格偏离
- 动作错误
- 结果不可用

#### 3. 备注输入
- 最多 200 字
- 可选

### 底部按钮
#### 普通标注场景
- 取消
- 保存标注

#### 下载拦截场景
- 取消
- 保存并下载

## 八、再生成逻辑

当用户点击“再生成”时：

- 新生成结果视为新结果
- 不继承旧结果的标注状态

也就是说：
- 原卡片保留原标注
- 新卡片默认为：
  - `pending`

## 九、当前数据字段

当前 `Generation` 已支持以下标注字段：

- `usability_status`
- `usability_reason_tags`
- `usability_note`
- `usability_marked_at`

定义位置：
- `src/types/index.ts`

## 十、数据库字段

Supabase migration：
- `supabase/migrations/20260421103000_add_usability_annotation_fields.sql`

新增字段：

- `usability_status text not null default 'pending'`
- `usability_reason_tags jsonb not null default '[]'`
- `usability_note text`
- `usability_marked_at timestamptz`

并增加约束：
- `pending`
- `usable`
- `optimizable`
- `unusable`

## 十二、当前持久化状态

当前代码已经补了数据层支持：

- `src/lib/supabase.ts`
  - 已补标注字段默认值
  - 已补 `updateUsabilityAnnotation()`

说明：
- 历史记录与资产页在保存时都会真正调用 `updateUsabilityAnnotation()`
- 标注结果会写入数据库
- 刷新页面后状态不会丢失

## 十三、当前版本的产品价值

可用性标注能力带来的价值：

1. 把“用户对结果质量的主观判断”沉淀下来
2. 下载行为从“无反馈”变成“有准入门槛”
3. 列表页可快速扫读哪些结果已经处理过
4. 方便后续扩展：
   - 质量统计
   - 标注分析
   - 结果筛选
   - 可用率统计
   - 训练数据沉淀

## 十五、验收标准

该功能可以认为完成，当满足：

- 历史记录中可见标注状态
- 资产页中可见标注状态
- 点击 `去标注 / 修改` 能打开弹层
- 未标注点击下载会被拦截
- 标注完成后卡片状态即时变化
- 下载拦截场景中，保存后能继续下载
- 数据结构和 migration 已补齐
- 标注刷新后不丢失
- 资产页可按标注状态筛选
